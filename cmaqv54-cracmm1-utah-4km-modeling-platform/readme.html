File Format

The model input are stored as compressed netcdf/hdf5 formatted files using I/O API data structures (https://www.cmascenter.org/ioapi/). Information on the model projection and grid structure is contained in the header information of the netcdf file. The netcdf files can be opened and manipulated using I/O API utilities (e.g. M3XTRACT, M3WNDW) or other software programs that can read and write netcdf formatted files (e.g. Fortran, R, Python).

Emissions Inputs

  • Anthropogenic emissions: The 2017 emissions modeling platform was used for most emissions source categories.
  • Lightning NO: lightning NO emissions were not used since simulation was done for a winter period
  • Biogenics: BEIS model run online in CMAQ with BELD4 land use data

    • Meteorology Inputs

  • WRF Model: Weather Research Forecast model, version 4.6.0 (WRF v4.6.0) simulated meteorology for 2017.
    • #Meteorology: /work/MOD3DEV/sgq/conus/UTAH/4km_UTAH/met_rob/
  • NLCD40 land use data (2011 version)
  • WRF post-processing: Meteorology-Chemistry Input Processor (MCIP) v5.3.3
